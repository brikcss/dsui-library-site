Vertical Rhythm is a principle of good typography which fosters repetation and familiarity, and creates a layout that is more balanced, beautiful, and readable. DS UI's Vertical Rhythm provides an easy way to apply and manage vertical rhythm in any app.

To see an example of what vertical rhythm is:

Notice how everything has consistent dimensions and spacing: font, line height, element heights, margins, padding, and so forth. Even elements which are completely disconnected from each other are "in rhythm". That's Vertical Rhythm.

Install

It is strongly encouraged to install vertical rhythm with DS UI Core:

npm install @directscale/core --save-dev

You may install vertical rhythm on its own:

npm install @directscale/rhythm --save-dev

Setup

  1. Include _rhythm.abstract.scss in your SASS build.
  2. Initialize vertical rhythm styles by doing one of the following:
    • Basic: Include _rhythm.init.scss into your SASS build.
    • Advanced: For greater customization, use _rhythm.init.scss as an example to initialize vertical rhythm on your own.

Basic Usage

Definitions:

Dimensional properties
CSS properties which apply height, width, margin, padding, line-height, border, or any other property which affects layout dimensions.
1rem
One (1) line in the vertical rhythm grid.

Rhythm Rules

In order to keep the vertical grid "in rhythm", it is important to strictly follow these rules:

  1. Always use rem units to apply dimensional properties.
  2. Apply rem values, if at all possible, in whole numbers. As necessary, rem values may also be applied in halves (x.5rem), or in rare cases in quarters (x.25rem, x.75rem). It is strongly discouraged to avoid any other fractional rem value since it will break the vertical rhythm.
  3. There may be rare cases where it's necessary to break the rhythm. Perhaps an element needs a 1px border, or an image needs a specific height in pixels. In such cases, follow these guidelines:
    • Ask yourself, is this really necessary? If at all possible, avoid breaking the vertical grid.

    • If you must temporarily break the rhythm, follow this rule to bring the grid back in rhythm:

      The sum of all dimensional property values for a given element, converted to rems, must be an integer.

    • A couple of examples to illustrate:

      // This pixel border may throw the vertical
// rhythm off by 2px (1px for border-top and
// border-bottom). Using the calc() function to
// readjust another vertical dimension by 2px
// will put the grid back in rhythm.
.pixel-border {
	border: 1px solid hsl(0, 0, 80%);
	height: calc(3rem - 2px);
}

// Let's assume this image must be 300px, and
// that our vertical rhythm (one vertical line)
// is set to 32px.
// A height of 300px = 9.375rem + (0.3125rem * 2)
// equals 10rem. With a 300px tall image, we're
// still "in rhythm"!
.pixel-image {
	height: 300px;
	margin-bottom: 0.3125rem;
	margin-top: 0.3125rem;
}
